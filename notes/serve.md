# `seek serve` — HTTP API

`seek serve` exposes the same search/fetch/crawl operations as the CLI over a
small JSON HTTP API, backed by the shared provider factory and `auto` failover.

## Endpoints

| Method | Path            | Auth | Body / notes |
|--------|-----------------|------|--------------|
| POST   | `/search`       | yes  | `{"query","provider?","range?","start?","end?"}` → `{"results":[...]}` |
| POST   | `/fetch`       | yes  | `{"url","provider?","format?"}` → `FetchResult` |
| POST   | `/crawl`        | yes  | `{"url","provider?"}` → `CrawlResult` |
| GET    | `/healthz`      | no   | liveness, returns `ok` |
| GET    | `/openapi.json` | no   | OpenAPI 3.0 spec |
| GET    | `/docs`         | no   | Swagger UI (loads `/openapi.json`) |

`start`/`end` are `DD/MM/YYYY`; `range` is "last N days". Dates are parsed by the
shared `buildTimeRange` (same semantics as the `search` CLI flags).

## Concurrency

`net/http` serves every request in its own goroutine, so the API is concurrent
by default. The operation runners (`opSearch`/`opFetch`/`opCrawl` in `ops.go`)
only *read* the factory, which is built once in `main()` and never mutated
afterward, so concurrent requests are safe. Each `auto` call builds its own
failover chain instance, so per-request attempt state never collides.

## Security

- Default bind is `127.0.0.1:8787` (loopback).
- `--token` (or `SEEK_SERVE_TOKEN`) requires `Authorization: Bearer <token>` on
  the operation endpoints; comparison is constant-time (`crypto/subtle`).
- With **no token the API is unauthenticated** — anyone who can reach the
  address can spend provider keys. seek logs a warning at startup, and an extra
  warning if a tokenless server is bound to a non-loopback address.
- `/docs` and `/openapi.json` are intentionally public (no secrets); only the
  operation endpoints are gated.
- Request bodies are capped at 1 MiB and reject unknown JSON fields.

## Status codes

`400` invalid request (bad JSON, missing field, bad date range) · `401`
unauthorized · `502` provider error · `200` success.

## Lifecycle

Graceful shutdown: when the command's context is cancelled (e.g. SIGINT) the
server is given 5s to drain in-flight requests via `http.Server.Shutdown`.

## ponytail notes

- Pure stdlib (`net/http`, `crypto/subtle`); no web framework.
- The OpenAPI spec is a hand-maintained JSON string in `swagger.go`, not
  generated by reflection — it's small and changes rarely. Ceiling: keep it in
  sync with the handlers by hand; revisit if the surface grows.
- Swagger UI is loaded from a CDN, so `/docs` needs internet on the viewer's
  side; the raw spec at `/openapi.json` is always self-contained.
